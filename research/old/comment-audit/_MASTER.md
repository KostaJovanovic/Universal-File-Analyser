# Comment audit - master fix list

Audit of every code/markup/CSS comment across the whole site (117 JS files + 7 HTML
pages + serve.py + sw.js + 6 tools/*.mjs + 2 CSS files), checking each comment for
**factual correctness** against the code it describes. Style/grammar/spelling were
out of scope.

**Result: 22 issues across 11 files.** The other ~100 files were clean. Per-group
detail lives alongside this file (`core-app.md`, `games.md`, `parsers-*.md`,
`renderers-*.md`, `html-tools.md`, `css.md`); this is the actionable rollup.

Each item: `file:line` - what's wrong -> the corrected comment.

---

## High value (stale facts that mislead future edits)

### 1. patch.html:107 - wrong constant name
`RELEASE in save.bat` -> the constant is **`RELEASES`** (plural; `set RELEASES=29,60,100`).
**Fix:** "... see RELEASE_COMMITS in app.js + **RELEASES** in save.bat ...".

### 2. patch.html:109 - stale "future 2.0"
"To crown a future 2.0, add that commit number to RELEASE_COMMITS." -> 2.0 and 3.0
already exist (`RELEASE_COMMITS = [29, 60, 100]`). Next crownable is **4.0**.
**Fix:** "To crown a future **4.0**, add that commit number to RELEASE_COMMITS."

### 3. assets/js/core/app.js:8-14 - stale versioning example
Comment still calls 3.0 a "future" release and only walks 29/60, but
`RELEASE_COMMITS = [29, 60, 100]` already crowns 3.0 (live era is 3.x).
**Fix:** "... commit 29 reads '1.0', commit 60 reads '2.0', and commit 100 reads
'3.0' (with 101 -> '3.01'). To crown a future 4.0, append its commit number here
(keep the list sorted ascending, and mirror the RELEASES constant in save.bat)."

### 4. atari.html:84 - footer marker claims generation that doesn't happen
`<!-- FOOTER:START - generated by tools/stamp-footer.mjs ... do not hand-edit -->`
but atari.html is **not** in stamp-footer.mjs `PAGES`, so the footer is never
stamped and has drifted stale (old "48/72/310 MB" tiers, "1017+ formats", missing
dependency rows).
**Fix:** either add `atari.html` to `PAGES` in stamp-footer.mjs (makes the claim
true and re-syncs the footer), or drop the "generated by ... do not hand-edit"
wording from atari's marker. *(Note: this is also a real content-staleness bug on
that page, not just a comment problem.)*

## Game logic comments (wrong numbers / swapped facts)

### 5. assets/js/games/boss.js:3-4 - wrong boss waves
"serpent on wave 7, megastructure on wave 10" -> code (`bossTypeForWave`/`isBossWave`,
and the file's own line 20) uses **8** and **11**.
**Fix:** "mothership on wave 5, serpent on wave **8**, megastructure on wave **11**".

### 6. assets/js/games/boss.js:7 - wrong unlock description
"unlocks the optional Wave 10 start" -> beating a boss reveals the start-wave picker
whose ceiling is `maxStartWave()` (= `bestWave - 2`), not a fixed Wave 10.
**Fix:** "the first boss beaten unlocks the optional start-wave picker".

### 7. assets/js/games/boss.js:535 - wrong UFO cap
"capped at 4 of its own alive at once" -> code checks `< 6` (line 539).
**Fix:** "...capped at **6** of its own alive at once".

### 8. assets/js/games/ufos.js:2-4 - swapped colours
"teal reward saucer ... magenta ambient escort" -> `UFO_REWARD_COLOR = '#ff4dd2'`
(magenta), `UFO_AMBIENT_COLOR = '#56d4dd'` (teal).
**Fix:** "a **magenta** reward saucer (destructible, drops a power-up) and a **teal**
ambient escort (indestructible, ...)".

### 9. assets/js/games/state.js:33 - "half" should be "minus 2"

**Fix:** "your best-ever wave minus 2, at least 1."

### 10. assets/js/games/asteroids.js:367-368 - same "half" mismatch
"the ceiling is half your best-ever wave (maxStartWave)" -> `maxStartWave()` returns
`bestWave - 2`.
**Fix:** "the ceiling is your best-ever wave minus 2 (maxStartWave)".

### 11. assets/js/games/leaderboard.js:168 - prefill never expires
"prefill with the name used in the last day" -> `rememberedName()` has no time check
(its own line 19 says "never expires").
**Fix:** "prefill with the remembered name (kept forever)".

## Binary-parser comments (wrong offsets / magic bytes)

### 12. assets/js/parsers/parsers-disk.js:961 - mislabelled EFI FV field
"header checksum + ext offset packed; skip" -> offset 44 is `Attributes` (u32) per
UEFI spec; Checksum is at 50, ExtHeaderOffset at 52.
**Fix:** "skip Attributes (u32 at offset 44); header length/checksum/ext-offset follow".

### 13. assets/js/parsers/parsers-gaming.js:1024 - wrong CHD v5 hunkbytes offset
`r.seek(0x54); // hunkbytes (v5)` -> CHD v5 `hunkbytes` is a u32 at **0x38** (56);
0x54 is inside the SHA-1 area. *(Possible real bug, not just the comment - verify
the seek.)*
**Fix:** "`r.seek(0x38); // hunkbytes (v5, u32 at offset 56)`".

### 14. assets/js/parsers/parsers-gaming.js:1572 - wrong VCDIFF 4th byte
"0xD6 0xC3 0xC4 'C' (xdelta3) then a version byte" -> self-contradictory; magic is 3
signature bytes (V|80 C|80 D|80) + a version byte (0x00). 4th byte is not 'C'.
**Fix:** "VCDIFF magic: 0xD6 0xC3 0xC4 (V|80 C|80 D|80) then a version byte (0x00)."

### 15. assets/js/parsers/parsers-gaming.js:256/258 - +512 offset never inspected
Comments describe checking both raw and +512 (0x300) Genesis copier offsets, but the
code only tests `SEGA` at 0x100, so copier-headered ROMs aren't detected. *(Comment
overstates the code; could also be a coverage bug.)*
**Fix:** "'SEGA' sits at 0x100 in a plain de-interleaved ROM; copier-headered ROMs
(SEGA at 0x300) are not detected here."

### 16. assets/js/parsers/parsers-audio.js:1058 - self-contradictory FAR magic
"Magic: 0xFE 'FAR' ... actually 'FAR\xFE'" -> the two halves disagree and the code
tests `FE` then "FAR". Farandole magic is "FAR" + 0xFE (`46 41 52 FE`).
**Fix:** "Magic: 'FAR' + 0xFE (bytes 46 41 52 FE)." *(and revisit the byte-order test
in code).*

## Renderer / core comments

### 17. assets/js/renderers/video.js:1294 - "uncompressed PCM" too broad
"Audio sample-entry fourCCs that are uncompressed PCM" -> the set includes `ulaw`/
`alaw` (companded, not uncompressed) and `fl32`/`fl64` (float).
**Fix:** "Audio sample-entry fourCCs the browser can't play in <video> - uncompressed
PCM (twos/sowt/lpcm/in16/in24/in32/raw /NONE/fl32/fl64) plus companded mu-law/A-law."

### 18. assets/js/renderers/proprietary-formats.js:374/389 - nsf misfiled
The `// ... parsers-email.js` section header encloses `nsf` (NES Sound Format) which
uses `chunk: 'audio'`.
**Fix:** move the `nsf` entry into the audio section (near `nsfe` ~line 767) so the
email-chunk header no longer wrongly claims it.

### 19. assets/js/core/binutil.js:51-52 - "clamped" but isn't
"clamped to a safe JS integer" -> `u64num()` returns `Number(v)` in **both** ternary
branches; no clamping.
**Fix:** "Number from a 64-bit unsigned. Converted to a JS Number, which loses
exactness above 2^53 - fine for file sizes / sample counts where that doesn't matter."

### 20. assets/js/renderers/audio.js:270 - dangling `dbFloor` reference
Doc mentions a `dbFloor` sensitivity param that `specStats(spec)` doesn't take, and
the sentence trails off.
**Fix:** drop the dangling `dbFloor` clause: "Scan a computed spectrogram for
at-a-glance stats. All levels are signal-relative ...".

### 21. assets/js/renderers/audio.js:264/270 (loudestMoment) - last-block centre
`time: (bestStart + block/2)/sampleRate` uses the nominal block size, so the final
(possibly short) block's reported centre can overshoot its true midpoint. Low severity.
**Fix:** note the centre uses the nominal block size (last short block's centre is
approximate).
